Functions

Just like for loops, functions allow us to do a task using a reusable block of code with different arguments.

It also allows us to develop with a DRY methodology:

Don't Repeat Yourself!

This is absolutely huge! Think about keeping your code "DRY" and you'll notice how nicely neat and organized things become.

Function definition syntax: function funcName ( argument, arg2, ... ,argN ) { ... }

Function call syntax: funcName ( myArgs );

Let's play with functions! Think about how functions make certain tasks easier. They also make our code neater! 

/* Basic function with no return value
   (This is a somewhat contrived example): */
function sayHi(greeting){
  alert(greeting);
}

// Small function with no arguments to flip a coin:
function coinToss(){
  return Math.round(Math.random());
}

/* Our arguments can have whatever type we'd like.
   Notice how here, we loop through whatever array we're
   given and never have to worry about what the name of
   the variable is.

   Take an array of strings, return a sentence: */
function arrayToString(words){
  var sentence = "";

  // Loop through words using nice "for each" shorthand
  for(var i in words){
    if(i < words.length-1)
      sentence += words[i] + " ";
    else
      sentence += words[i] + ".";
  }

  // Return full sentence. This is critical!
  return sentence;
}

/* We can add as many arguments as we'd like.
   This is a rewrite of the Math.pow() function. */
function pow(base, exponent){
  var expValue = base;

  for(var i = 1; i < exponent; i++)
    expValue *= base;

  return expValue;
}

Arrays vs. Objects

A Javascript object is the most general variable type. Arrays are objects with some special syntax and specific methods. There are also many other different types of objects, but we'll just be focusing on the most versatile type.

The second, however, uses two separate arrays that relate to each other and are difficult to combine. How would we describe an association between these two arrays? Is there a better way?

In short, yes! Objects allow us to store data associatively. Instead of being limited to numbered indices, we can actually give data names. Even better, we can refer to each property using array or dot-syntax.

Scope and Callbacks

Javascript has rules for where variables exist and how they can be accessed; this changes depending on where they are defined, if they're nested in brackets, as function arguments, etc. Where a variable can be accessed is referred to as its scope.

Scope is a ridiculously important concept in Javascript!

The Breakdown

  • Variables defined within scripts tags or otherwise outside of all other functions/loop bodies/brackets: global variables
  • Variables defined just within functions/loop bodies/brackets: local variables. Visible only within the brackets
  • Global variables can be accessed within functions as well as outside. Belong to global scope
  • Local variables disappear after functions return and loops break
