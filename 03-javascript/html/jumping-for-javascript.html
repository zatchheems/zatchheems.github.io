For Loops

Sometimes, it can be tedious to look at or modify many things sequentially.

It can also be a huge time-waster; if we're doing the same thing for multiple elements in an array, for example, it would be nice to not have to copy and paste the same line over and over again. We can automate repeditive tasks with loops.

Enter the for loop: it's a super common way to run the same line(s) of code as many times as we'd like, using a counter to access, output, or manipulate data.

Syntax:

for ([counter]; [exit condition]; [iterator]){...} 
// Output each value of the counter

console.log("Even numbers from 0 to 20: ");

for(var i = 0; i <= 20; i += 2){
  console.log(i);
}

var flowers = [ 'Daffodil', 'Hyacinth', 'Orchid', 'Tulip', 'Hosta' ];

// Example, using flowers array declared earlier.
//  1. Declare a new variable as a counter
//  2. Loop until the counter = length of array
//  3. Increment counter after loop

for(var i = 0; i < flowers.length; i++){
  console.log(flowers[i]+"s? Those are dope.");
}

Browser Events

What happens when you...

  • move your cursor across the window?
  • scroll within the window?
  • click within the viewport?
  • press any key on your keyboard?
  • hover over an element?

Your browser listens to all of the input that you give it, keeps track of it, and can be programmed to respond accordingly. A lot of elements have default ways to respond to these events.

There a couple of ways to get your browser to listen and respond to events, and you can add event listeners to different elements.

The most common way to select an element—a DOM node—is document.getElementById("elementName"). Assign this to a variable and we can do something like the following:

// Look in the document for an element with ID #myElem.
// Notice that we don't use the pound sign.

var myElem = document.getElementById("myElem");

// We can listen for clicks on the element like so:

myElem.onClick = function(event){
  // I got clicked on!
  // Do a thing!
}

// You can also use a more general syntax:

myElem.addEventListener("click", function(event){
  // Same as above, but more general.
  // We can replace "click" with other events, like
  //  "hover" or "keyPressed".
});

// Here, we want the key pressed anywhere on the page:

document.onkeyup = function(event){
  // We can see the key that got pressed and use it!
  console.log(event.key);
}

We can modify DOM nodes on the fly; properties such as textContent can be changed for elements that can hold text, styles can be changed, etc. Pretty much all of the information a DOM node has is editable!

Quick demo:

// Store button, number of button clicks
 var numClicks = 0;
 var button = document.getElementById("click-me");

// Add an event listener for clicks on button
button.onclick = function(event) {
if (numClicks === 0){
    button.textContent = "Alas! I am clicked!";
    ++numClicks;
  } else if(numClicks === 1) {
    button.textContent = "You continue to click me! Why?";
    ++numClicks;
  } else {
    button.textContent = "Have you no shame?! You clicked me "+
      (numClicks+1)+" times already!";

    ++numClicks;
  }
};
