NYT Article Search Application

As a refresher, let's revisit the New York Times Article Search Application (kind of a mouthful, it'll continue to be NYT from here on out).

Remember that the most important functionality is that it allows you to search their article API—here's that API in action.

Also remember: this is a hard activity to do completely right. It requires you to code quickly, efficiently divide tasks, and deal with bugs. It's also great for gaining mastery over these topics.

You should be partnering in sub-pairs, where one pair does UI and the other does API calls. Take honest stock of your abilities and focus on what you're comfortable doing.

Why this specific format?

A specific developer (or team of developers) is often designated to work on a specific feature. When working as separate teams, developers need to learn to communicate needs and expectations with other devs working on related features (i.e. data teams should expect text inputs from the UI team).

Without further ado, let's get into it!

Group activity!

Phase I: Prep

📐 UI Team:

  • Create the layout for the design. Feel free to use Bootstrap or to do something more customized of your own.
  • Make sure to organize your code so as to have the necessary IDs.
  • Incorporate text boxes for capturing User Input. Then research how to retrieve the input values in JavaScript.

🛠 Data Team:

  • Do preliminary research on the API.
  • Register for an API Key if you have not already done so.
  • Understand what format the URL should look like to make an Article Call.
  • Experiment with console logging various fields.

Group activity!

Phase II: Logic

📐 UI Team:

  • Begin creating basic click events. Register the submit button.
  • Get the data from the text-boxes and store as variables.
  • Experiment with creating content regions for where the article will go.

🛠 Data Team:

  • Create the AJAX call needed to retrieve data then console.log all of the relevant fields.
  • Incorporate various “optional parameters” (hard code these in initially).
  • Take note of various “bugs” that appear with certain searches.

⚙ Everyone:

  • Display the HTML content!

Group activity!

Phase III: Bugfixes

📐 UI Team:

  • Continue polishing the display of content in the HTML.

🛠 Data Team:

  • Put in a hard-effort to deal with bugs. How can you handle missing fields?
