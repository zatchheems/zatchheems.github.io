Mern Demo

Let's begin today's class with a demo of a full Mern app. As a general reminder: Mern stands for:

  • MongoDB
  • Express
  • ReactJS
  • NodeJS

Since beginning to learn React ,we've only been working with front-end code. We haven't set up a database, or an Express server; we've just been working on the front-end with React. Though we have been using a dev server with Create React App, this server exists simply for live reloading and compiling. Recall that everything gets transpiled and minified into a bundle.js—this is what gets deployed. In most real applications, we'll also need a production quality server such as Express for connecting to a database or setting up API routes.

You will be provided with a Create React App Express boilerplate that you can use for homework or projects. There's no need to completely understand every aspect of the code in order to be productive, but we're going to walk through an example to give you a working high level understanding.

Let's take a look at 07-Ins_Mern. Within the client folder, there's something weird here. Unusually, we have not one, but two package.json files, each with different sets of dependencies. Running yarn install will install two different servers to be used together. We'll also want to pay attention to the proxy setting in Create React App and pore through the two package files.

We have our usual yarn start to run the app, and also yarn build for a static version of our React app that can be deployed.
