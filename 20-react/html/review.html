Requirements

Here are the requirements for the final project:

  • Must use React, Vue, or Angular in some way (even if minimal)
  • Must use a Node and Express Web Server
  • Must be backed by a MySQL or MongoDB or ORM-plugin-compatible (PostgreSQL, Redis) database with a Sequelize or Mongoose ORM
  • Must have both GET and POST routes for retrieving and adding new data
  • Must be deployed using Heroku (with Data) or PaaS of your choice
  • Must utilize at least two libraries, packages, or technologies that we havenâ€™t discussed
  • Must have a polished frontend / UI
  • Must have folder structure that meets MVC Paradigm
  • Must meet good quality coding standards (indentation, scoping, naming)
  • Must not expose sensitive API key information on the server

These are somewhat flexible! Do what you want. Talk to us and get our input. If you have an idea, run with it! This is the time to prove yourself.

Deliverables:

Week 1: Action Plan

A detailed plan of action with the following:

A functioning minimal viable product—i.e., a prototype version of your app that shows the intended function. This should be as close to a working app as possible and include:

  • A screenshot of your Project Management Board of choice that shows breakdown of tasks assigned to group members with a schedule. This should be updated to reflect remaining priorities and completed tasks.

A significantly more polished version of the app. This should literally seem like an app that you would submit.
