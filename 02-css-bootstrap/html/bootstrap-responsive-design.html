Sketching

Every site is defined by its look. Having a basic sketch of how you want your site to look is incredibly important! Instead of just messing around until something looks good, you can work off of a detailed sketch and really try to nail the look you want without guessing and checking.

Nav bars, main content, images, footers, etc. should all be considered. They don't have to be super well-drawn (and you can make sketches on your machine if you want!) but should give a good idea of how you want things to be laid out.

It also helps to design elements on a grid. Grids serve to align items in an aesthetically pleasing and logical way—e.g. aligning content with the header, making a large block of content scroll smoothly into a few columns of content, etc.

Facebook, LinkedIn, Instagram and other social media sites as well as many news sites champion the usage of a grid layout: content is aligned via clearly-set "lines". Content, containers, images, etc. align in an aesthetically pleasing way.

A few wireframing tools:

  • Balsalmiq — https://balsamiq.com/
  • Framebox — http://framebox.org/
  • Illustrator/Photoshop with grid tools (or without, but turn the grid view/align on!)
    • 960 Grid System — https://960.gs/
    • GuideGuide (paid w/free trial) — https://guideguide.me/
  • By hand in a notebook

Google is your friend when learning how to design by grid. There's also http://www.designbygrid.com/.

A Bootstrap grid is comprised of:

  • Containers: wrap around entire grid
  • Rows: defined first, contain columns

Mobile Responsiveness

Media queries

Media queries define how CSS styles are applied based on the characteristics of the device viewport. They change how a page is displayed based on width, height, orientation, and media type. Bootstrap uses these extensively!

Media queries must be declared at the very bottom of your stylesheet.

Viewport

The viewport refers to the display being used to view a website. Mobile devices with small screens and high resolution render at desktop sizes. We can change this with a meta tag that tells the page to render based on the current viewport:

<-- HTML -->
<meta name='viewport' content='width=device-width, initial-scale=1'>

/* CSS */
@media screen and (max-width: 900px){
  /* ... */
}

@media screen and (max-width: 500px){
  /* ... */
}

Try using the developer tools to emulate different screen sizes and devices on your favorite sites to see how they respond.
